Цель: описать базовые свойства и различия 7 источников котировок из `ods.parquet`:

- `B365` (`B365W/B365L`)
- `EX` (`EXW/EXL`)
- `LB` (`LBW/LBL`)
- `PS` (`PSW/PSL`)
- `Max` (`MaxW/MaxL`)
- `Avg` (`AvgW/AvgL`)
- `BFE` (`BFEW/BFEL`)

Под “patтернами” ниже понимаются:

- **покрытие** (сколько матчей имеют обе котировки W/L)
- распределения odds / implied probability
- **маржа** (overround): `1/W + 1/L`
- **согласованность** между буками: корреляция и MAE по implied probability

## 1) Подготовка: implied probability и overround

Для каждой пары odds (W/L) считаем:

- `pW_raw = 1/W`, `pL_raw = 1/L`
- `overround = pW_raw + pL_raw`
- `imp_p1 = pW_raw / overround` (нормализованная implied probability)

Интерпретация:

- `overround > 1` — положительная маржа (типичный случай)
- `overround ≈ 1` — “почти честные” котировки
- `overround < 1` — арбитраж/аномалия/шум данных

## 4) Согласованность implied probability между букмекерами

Сравниваем по `imp_p1` (нормализованная implied probability). Для каждой пары букмекеров считаем:

- `n_overlap` — количество матчей, где у обоих есть W/L
- `corr_imp_p1` — корреляция `imp_p1`
- `mae_imp_p1` — средняя абсолютная разница `|imp_p1_a - imp_p1_b|`

## 5) Практические рекомендации для моделей

- Для **baseline implied probability** лучше опираться на источники с высокой полнотой и предсказуемой маржой:
  - `PS`, `B365`, `Avg`, `Max`.
- Если использовать несколько буков одновременно:
  - обязательно привести к implied probability (нормализовать маржу),
  - аккуратно работать с пропусками (не смешивать буки с coverage 5% и 99% без контроля).
- По текущим цифрам `EX` имеет заметно более высокий overround и низкое покрытие: это кандидат на “шумный” источник.
